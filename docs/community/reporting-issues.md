## Reporting Issues

If you encounter bugs, errors, or unexpected behavior in SHKeeper, you can report them to ensure they are addressed promptly by the development team.

### How to Report an Issue

1. **Check Existing Issues**  
   Before creating a new issue, review existing reports on GitHub to avoid duplicates:  
   [SHKeeper GitHub Issues](https://github.com/vsys-host/shkeeper.io/issues)

2. **Create a New Issue**  
   Provide clear and detailed information, including:
   - **Title:** A concise summary of the problem  
   - **Description:** Detailed explanation of the issue  
   - **Steps to Reproduce:** Exact steps to trigger the bug  
   - **Expected Behavior:** What you expected to happen  
   - **Actual Behavior:** What actually happened  
   - **Environment:** Server OS, SHKeeper version, Kubernetes/Helm setup, browser, etc.  

3. **Attach Logs or Screenshots**  
   Include relevant logs, error messages, or screenshots to help the developers understand the problem.

4. **Submit the Issue**  
   Once all details are included, submit the issue on GitHub:  
   [https://github.com/vsys-host/shkeeper.io/issues](https://github.com/vsys-host/shkeeper.io/issues)

### Tips

- Be specific and concise to help the team reproduce the issue faster.  
- Use proper formatting (Markdown) to organize your report.  
- If the issue is urgent or security-related, consider contacting SHKeeper support directly via email: **support@shkeeper.io**.
